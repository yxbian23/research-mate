# Cross-Validation Methods for Homework Solutions

作业解答的交叉验证方法，确保答案的正确性。

---

## 核心原则

**为什么需要交叉验证？**

1. 防止计算错误
2. 发现逻辑漏洞
3. 增强答案可信度
4. 学习多种解题思路

**何时需要验证？**

- 复杂的数值计算
- 多步推导的最终结果
- 关键的中间结论
- 反直觉的答案

---

## 验证方法分类

### 1. 数值验证 (Numerical Verification)

**方法**: 使用不同的计算方式重新得到结果

**适用场景**: 计算题、数值答案

**示例**:

```
问题: 计算混合策略纳什均衡

方法1 - 无差异条件:
设 Player 1 的混合策略为 (p, 1-p)
Player 2 选择 L 和 R 的期望收益相等:
3p + 0(1-p) = 0p + 1(1-p)
3p = 1 - p
p = 1/4

方法2 - 支付矩阵直接计算:
令 U₂(L) = U₂(R)
代入具体数值计算...
p = 1/4 ✓

验证通过: 两种方法得到相同结果
```

### 2. 边界检验 (Boundary Testing)

**方法**: 检验极端情况或边界条件下的结果

**适用场景**: 含参数的公式、优化问题

**示例**:

```
问题: 证明对所有 n ≥ 1，有 Σᵢ₌₁ⁿ i = n(n+1)/2

边界检验:
- n = 1: LHS = 1, RHS = 1(2)/2 = 1 ✓
- n = 2: LHS = 1+2 = 3, RHS = 2(3)/2 = 3 ✓
- n = 10: LHS = 55, RHS = 10(11)/2 = 55 ✓

边界检验通过
```

### 3. 维度分析 (Dimensional Analysis)

**方法**: 检查公式两边的量纲/单位是否一致

**适用场景**: 物理问题、经济模型、带单位的计算

**示例**:

```
问题: 验证公式 E = mc²

维度分析:
- E (能量): [M L² T⁻²]
- m (质量): [M]
- c (光速): [L T⁻¹]
- mc²: [M] × [L T⁻¹]² = [M L² T⁻²] ✓

维度匹配
```

### 4. 特例验证 (Special Case Verification)

**方法**: 用简单的特例验证一般性结论

**适用场景**: 一般性定理、递推公式

**示例**:

```
问题: 证明 (A ∪ B)ᶜ = Aᶜ ∩ Bᶜ (De Morgan's Law)

特例验证:
设 U = {1,2,3,4,5}, A = {1,2}, B = {2,3}

LHS: A ∪ B = {1,2,3}
     (A ∪ B)ᶜ = {4,5}

RHS: Aᶜ = {3,4,5}
     Bᶜ = {1,4,5}
     Aᶜ ∩ Bᶜ = {4,5} ✓

特例验证通过
```

### 5. 反例检验 (Counterexample Testing)

**方法**: 尝试找反例来检验结论

**适用场景**: 证明题、命题判断

**示例**:

```
问题: "所有连续函数都可微" - 判断真假

反例检验:
f(x) = |x| 在 x = 0 连续但不可微

结论: 命题为假，存在反例
```

### 6. 对称性检验 (Symmetry Check)

**方法**: 利用问题的对称性验证答案

**适用场景**: 对称博弈、概率问题

**示例**:

```
问题: 在对称博弈中求纳什均衡

对称性检验:
如果博弈是对称的，纳什均衡应该也是对称的
即两个玩家的策略应该相同

验证: 如果得到 (p, 1-p) 和 (q, 1-q) 其中 p ≠ q
需要重新检查计算（因为对称博弈应有 p = q）
```

### 7. 逆向验证 (Reverse Verification)

**方法**: 将答案代回原问题验证

**适用场景**: 方程求解、优化问题

**示例**:

```
问题: 求解方程 x² - 5x + 6 = 0

解: x = 2 或 x = 3

逆向验证:
- x = 2: 4 - 10 + 6 = 0 ✓
- x = 3: 9 - 15 + 6 = 0 ✓

验证通过
```

## 不同题型的验证策略

### 证明题

| 验证方法 | 说明 |
|---------|------|
| 特例验证 | 用具体例子验证一般结论 |
| 反例检验 | 确保不存在反例 |
| 逻辑检查 | 检查每步推导的有效性 |
| 边界检验 | 检查边界情况 |

**检查清单**:
- [ ] 每个 "因为" 后面的理由都正确？
- [ ] 每个 "所以" 前面的条件都满足？
- [ ] 证明覆盖了所有情况？
- [ ] 结论与题目要求一致？

### 计算题

| 验证方法 | 说明 |
|---------|------|
| 数值验证 | 用不同方法重新计算 |
| 逆向验证 | 代回原式检查 |
| 维度分析 | 检查单位/量纲 |
| 数量级检查 | 结果数量级是否合理 |

**检查清单**:
- [ ] 计算步骤无遗漏？
- [ ] 符号运算正确？
- [ ] 最终答案代回原式正确？
- [ ] 答案数量级合理？

### 博弈论题

| 验证方法 | 说明 |
|---------|------|
| 最佳响应检验 | 验证是否为相互最佳响应 |
| 对称性检验 | 对称博弈的均衡应对称 |
| 支付验证 | 偏离均衡应导致收益下降 |
| 边界检验 | 检查概率是否在 [0,1] 内 |

**检查清单**:
- [ ] 对每个玩家，给定对方策略，当前策略是最佳的？
- [ ] 混合策略概率之和为 1？
- [ ] 所有概率都非负？
- [ ] 无差异条件满足（对于混合策略均衡）？

### 优化题

| 验证方法 | 说明 |
|---------|------|
| 一阶条件检验 | 梯度为零或 KKT 条件满足 |
| 二阶条件检验 | Hessian 的正定/负定性 |
| 可行性检验 | 解满足所有约束 |
| 边界检验 | 检查边界点 |

**检查清单**:
- [ ] 解满足所有约束条件？
- [ ] 满足必要的最优性条件 (FOC)？
- [ ] 满足充分的最优性条件 (SOC)？
- [ ] 与边界点比较，确保是全局最优？

## 常见错误类型

### 计算错误

- 符号错误 (正负号)
- 算术错误
- 代数化简错误
- 忘记约分

### 逻辑错误

- 必要条件当充分条件
- 遗漏情况
- 循环论证
- 跳步过大

### 理解错误

- 误解题意
- 混淆定义
- 使用错误的定理
- 条件不满足时使用定理

## 验证效率建议

1. **先粗后细**: 先做快速检查（量级、特例），再做详细验证

2. **关键点优先**: 优先验证最可能出错或最关键的步骤

3. **独立验证**: 尽量用不同于原解法的方法验证

4. **利用工具**: 简单数值计算可用计算器或代码验证

5. **记录过程**: 记录验证过程，便于发现问题

---

## 示例: 完整的验证流程

**问题**: 找出以下博弈的混合策略纳什均衡

|   | L | R |
|---|---|---|
| T | (2,1) | (0,0) |
| B | (0,0) | (1,2) |

**解答过程** (略)

**答案**: Player 1 选 T 的概率 p = 2/3，Player 2 选 L 的概率 q = 1/3

**验证流程**:

1. **概率范围检查**
   - p = 2/3 ∈ [0,1] ✓
   - q = 1/3 ∈ [0,1] ✓

2. **无差异条件验证**

   Player 1 的期望收益:
   - E[T] = 2×(1/3) + 0×(2/3) = 2/3
   - E[B] = 0×(1/3) + 1×(2/3) = 2/3
   - E[T] = E[B] ✓

   Player 2 的期望收益:
   - E[L] = 1×(2/3) + 0×(1/3) = 2/3
   - E[R] = 0×(2/3) + 2×(1/3) = 2/3
   - E[L] = E[R] ✓

3. **最佳响应检验**
   - 给定 q = 1/3，Player 1 对 T 和 B 无差异，任何混合都是最佳响应 ✓
   - 给定 p = 2/3，Player 2 对 L 和 R 无差异，任何混合都是最佳响应 ✓

4. **偏离检验**
   - 如果 Player 1 偏离到纯策略 T，收益仍为 2/3 (不会更好)
   - 如果 Player 1 偏离到纯策略 B，收益仍为 2/3 (不会更好)
   - 同理 Player 2 ✓

**验证结论**: 答案 (p=2/3, q=1/3) 通过所有验证，是正确的混合策略纳什均衡。
